### Описание работы прогнозной модели

Разработанная система прогнозирования страховых премий основана на двухэтапном подходе, учитывающем временную природу страховых данных.
Архитектура

1. Предобработка данных

    Очистка пропусков и выбросов с использованием статистических методов (квантильная фильтрация)

    Feature engineering: расчёт длительности полиса, возраста объекта страхования

    Категориальное кодирование: частотное кодирование для марки авто, Target для типа ТС и цели использования

    Коррекция данных на основе ассоциативных правил Apriori (выявление и исправление нелогичных комбинаций признаков)

2. Двухэтапное прогнозирование

*Этап 1*: Начальное прогнозирование (**Gradient Boosting**)

    Для первого полиса по каждому объекту (отсутствует история платежей)

    Используется градиентный бустинг с подбором гиперпараметров через GridSearch

    Прогноз формируется на основе характеристик объекта страхования

*Этап 2*: Последующее прогнозирование (**Нейронная сеть**)

    Для повторных полисов, где доступна история платежей

    Используется многослойный перцептрон с ранней остановкой

    Входные признаки дополняются: предыдущая премия и накопленная длительность страхования

    Учитывается временная динамика через кумулятивные характеристики

3. Управление состоянием

    Система хранит для каждого объекта последнюю премию и суммарную длительность страхования

    Это позволяет модели учитывать историю платежей при прогнозировании

    Состояние сохраняется на диск вместе с моделями

4. Разделение по типу страхования

    Отдельные модели для полисов с нулевой и ненулевой страховой суммой

    Учитывает принципиально разную природу этих типов страхования (ОСАГО vs КАСКО)

## Результаты

Модель обеспечивает временную консистентность прогнозов (не использует будущие данные для прогнозирования прошлого), корректно обрабатывает новые объекты и учитывает накопленную историю для существующих.

## Описание функционала run.py

Скрипт `run.py` предоставляет интерфейс командной строки для работы с системой прогнозирования страховых премий. Поддерживает два режима работы: инференс (прогнозирование) и обновление модели.

### Режимы работы

#### 1. Режим инференса (`-mode inference`)
```bash
python run.py -mode inference -file ./path_to_file.csv
```

**Назначение**: Прогнозирование страховых премий для новых данных.

**Логика работы**:
- Проверяет наличие столбца `PREMIUM` во входных данных
- **Если PREMIUM присутствует** (обучающие данные):
  - Данные добавляются в накопительный файл `data/training_data.csv`
  - Прогнозирование НЕ выполняется
  - Выводится сообщение о необходимости запуска обновления модели
- **Если PREMIUM отсутствует** (данные для прогноза):
  - Загружает обученную модель из `models/`
  - Выполняет прогнозирование
  - Сохраняет результат в файл с суффиксом `_predicted`
  - Выводит статистику прогнозов (мин, макс, среднее, медиана)

#### 2. Режим обновления (`-mode update`)
```bash
python run.py -mode update
```

**Назначение**: Переобучение модели на всех накопленных данных.

**Логика работы**:
- Загружает все накопленные обучающие данные из `data/training_data.csv`
- Выполняет полный цикл обучения (предобработка, подбор гиперпараметров, обучение)
- Сохраняет обученные модели в `models/`
- Выводит метрики качества обучения (RMSE)

### Примеры использования

```bash
# Первичное обучение (первичное)
python run.py -mode update

# Прогнозирование для новых данных без PREMIUM
python run.py -mode inference -file ./new_policies.csv

# Добавление новых обучающих данных (с PREMIUM)
python run.py -mode inference -file ./actual_premiums_2024.csv

# Переобучение модели на всех накопленных данных
python run.py -mode update
```
